Where the World Drinks Safely — SDG 6.1.1

This project models global inequalities in clean water access by mapping the United Nations Sustainable Development Goal indicator 6.1.1—the percentage of a population using a safely managed drinking water service. The workflow began in QGIS, where I joined national-level water data to country boundary shapefiles, filtered out tracking errors, and standardized the results.

The interactive version was built using Leaflet.js, incorporating custom styling, dynamic pop-ups, and an interactive legend to display global inequalities cleanly. From a cognitive perspective, using a structured color palette prevents visual overload, allowing viewers to immediately identify macro-regional disparities.

Kindergarten Accessibility in Utrecht by Bicycle

Spatial Question: How accessible are early childhood educational facilities across the municipality of Utrecht when using active bicycle transport, and where do infrastructural inequities occur?

Methods & Steps: I built a network dataset using real cycling pathways rather than simple straight-line buffers. Using vector network service area tools, I generated travel-time polygons (isochrones) at intervals of 4, 8, 12, 16, 20, and 24 minutes. I then ran a Vector → Analysis → Count Points in Polygon process, setting the isochrones as the base layer to calculate the exact number of kindergartens accessible within each time threshold across the municipal zones.

Findings & Spatial Patterns: The data revealed strong clustering within the urban core, where families can access multiple facilities in under 8 minutes. However, peripheral residential sectors showed distinct "service deserts," requiring over 20 minutes of travel time.

Neuroscience Link: Early environmental complexity and rapid access to social play spaces are vital catalysts for cognitive map formation and neurodevelopment in early childhood.

Flood Risk & Spatial Vulnerability Analysis

Spatial Question: Which urban catchment structures face the highest risk during extreme weather events, and how does topography dictate local water pooling?

Methods & Steps: This environmental model was constructed by analyzing Digital Elevation Models (DEMs). I calculated hydrological flow accumulations, surface slope gradients, and pooling constraints in QGIS. By combining these topographic features with municipal asset layers, I mapped out high-vulnerability risk profiles.

Findings & Regional Patterns: While many expect coastal flooding to be the primary threat in the Netherlands, this model highlights a different risk. In areas like the dunes of The Hague, the coastal barrier remains highly resilient. Instead, flooding here is largely internal, driven by heavy rainfall over saturated soils and high groundwater tables that pool in low-lying urban areas behind the dunes.

Coursework Report: Vegetation Indices & Environmental Metrics

Course: Geographic Information Science and Systems (UC-SCI-EARL5) | Geosciences, Utrecht University
Instructor: Britta Ricker, PhD

This independent research project analyzes urban expansion and environmental health in Valencia, Spain, from 2017 to 2024. The following report integrates satellite remote sensing data with spatial theory to assess environmental sustainability.

1. True Color Imagery Baseline (2017)

This map establishes a 2017 snapshot of Valencia using a True Color Composite (RGB Bands 4, 3, 2) from Sentinel-2 satellite data. To generate this, I adapted JavaScript scripts within Google Earth Engine to aggregate raw, open satellite data captured during the dry summer season, filtering out cloud cover to output a clear orthomosaic.

An orthomosaic is a uniform mosaic image compiled from overlapping satellite photos. It is geometrically corrected to remove camera tilt and topographic distortions, making it accurate enough to measure true distances like a map.

Neuroscience Link: True color maps mirror our visual system's trichromatic cone receptors, creating an intuitive mental map for immediate feature recognition.

2. True Color Imagery Comparison (2024)

This map shows Valencia in 2024, using the exact same summer timeline and 10m/pixel spatial resolution settings as the 2017 baseline. Comparing the two images highlights clear changes along the city's edges, documenting urban expansion and land use shifts over a seven-year period.

Why seasonality matters: Choosing the same month is essential when building orthomosaics. Comparing a winter scene with a summer scene introduces seasonal changes in foliage and sun angles, creating false trends in our change analysis.

View Full Screen ↗

3. Baseline NDVI Spectrum Analysis (2017)

This map displays the calculated Normalized Difference Vegetation Index (NDVI) for 2017, rendered using a Singleband Pseudocolor scale from -1 (unhealthy/water) to 1 (vibrant canopy). The calculation uses the formula:

NDVI = (B8 - B4) / (B8 + B4)

Where Band 8 is Near-Infrared (NIR) and Band 4 is Visible Red. This calculation works because healthy plant chlorophyll absorbs visible red light for photosynthesis but reflects near-infrared light back into space. By comparing these values, we can calculate a clear vegetation health score.

View Full Screen ↗

4. Comparative NDVI Spectrum Analysis (2024)

The 2024 NDVI layer maps out vegetation density across modern Valencia. This type of multispectral imagery is invaluable for urban planning. Because satellites record wavelengths beyond human vision, they allow us to monitor environmental changes that aren't visible from the ground.

This analytical approach directly supports UN Sustainable Development Goal 15 (Life on Land). It provides open, verifiable data to track urban expansion, protect surrounding ecosystems, and monitor city greening initiatives over time.

View Full Screen ↗

5. NDVI Environmental Change Delta Map

This final map shows net change over time, calculated via QGIS Raster Band Math:

NDVI 2017 - NDVI 2024 = Change Delta

Using a monochromatic scheme, this layout isolates environmental trends. Deep green shades represent biomass growth, while red tones reveal vegetation loss. This analysis helps differentiate actual urban expansion from temporary shifts caused by cloud artifacts or crop rotation cycles.

Supervised Pixel-Based Image Classification Pipeline

This project explores how computers process satellite imagery, showcasing the workflow of transforming raw data into automated land-cover maps using machine learning classifiers.

1. Raw Satellite Imagery Baseline

The starting point: raw, unprocessed satellite data. Sensors capture information across several bands of the light spectrum. Different surface features—like asphalt, water, and trees—reflect light in unique ways, providing the data blocks needed to distinguish them.

View Full Screen ↗

2. Spatial Resolution & Grid Pixel Analysis

This layer evaluates spatial resolution—the actual ground size represented by each pixel. Understanding pixel dimensions is essential because it sets the detail limits of our map, determining whether small features stay sharp or blur into surrounding pixels.

View Full Screen ↗

3. Localized Spectral Profile Diagnostics

This step charts spectral profiles—the unique light signature of different surface materials. Because objects reflect different amounts of light across separate color channels, plotting these values allows the computer to tell materials apart based on their light signature.

View Full Screen ↗

4. Unsupervised Pixel-Based Classification

In an unsupervised classification, a computer algorithm automatically groups pixels based entirely on their mathematical similarity. Without human guidance, the software finds clusters of matching values and groups them into raw data classes.

View Full Screen ↗

5. Unsupervised Object-Based Segmentation

Moving beyond single pixels, Object-Based Segmentation groups neighboring pixels with similar colors, textures, and shapes into distinct "objects." This method better mimics how human eyes see, grouping pixels into shapes like fields or building footprints before categorizing them.

View Full Screen ↗

6. Supervised Object-Based Classification

The final, most accurate step. By training the algorithm with real-world examples—manually pointing out known houses, water bodies, and forests—the computer learns to accurately categorize all the segmented shapes across the whole region, creating a clean land-cover map.
